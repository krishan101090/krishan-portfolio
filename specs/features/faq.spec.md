# Feature · FAQ (Rich-Snippet Optimized)

| Field       | Value                              |
|-------------|------------------------------------|
| Status      | Shipped                            |
| Component   | `components/FAQ.jsx`               |
| Data source | `profile.faqs[]`                   |

## 1. Intent

A visible, accessible FAQ whose content is **character-identical** to the
`FAQPage` JSON-LD schema — so Google shows rich Q&A results for queries
like "hire frontend architect" or "what is spec-driven development".

## 2. User story

As a searcher on Google, I want to see answers to common "hire this
engineer" questions inline in the SERP so I can decide to click through.

## 3. Scope

**In:** a list of questions, each expandable to reveal the answer, with
native HTML semantics (`<details>` / `<summary>`).
**Out:** search/filter within FAQ, user voting on answers, nested FAQs.

## 4. Data contract

```ts
type FAQ = {
  q: string;   // ends with '?'; ≤ 100 chars
  a: string;   // 1–3 sentences, plain prose; no HTML
};
// profile.faqs.length >= 3  (required for rich-result eligibility)
```

## 5. Behavior / states

| State       | Behavior                                                  |
|-------------|-----------------------------------------------------------|
| Default     | All questions collapsed.                                  |
| Expanded    | `<details open>`; answer revealed below with a fade.      |
| Multiple    | Multiple items may be open simultaneously — no accordion. |
| Deep link   | `#faq-<kebab(q)>` auto-opens that item on load.           |

## 6. Visual

- Each item: bordered row, left border turns accent on hover/open.
- `<summary>` uses mono font for the leading `?` token, body font for the
  question text.
- A chevron icon rotates 90° on open (pure CSS, respects reduced-motion).
- Answer paragraph in `--ink-1`, max 60ch.

## 7. Non-functional requirements

- **A11y:** use native `<details>`/`<summary>`; no ARIA needed.
  Keyboard: `Enter` / `Space` toggles, `Tab` moves to next item.
- **Perf:** zero JS (native disclosure).
- **SEO:**
  - Visible Q and A text must equal the JSON-LD `mainEntity[i].name` and
    `acceptedAnswer.text` exactly.
  - `FAQPage` schema generated by `lib/schema.js` is injected once at the
    page level; it is NOT duplicated per-item.

## 8. Acceptance criteria

- [x] **Given** 5 FAQs in data, **When** the page renders, **Then** 5
      `<details>` blocks are present in SSR HTML in authored order.
- [x] **Given** a URL with `#faq-how-do-you-charge`, **When** the page
      loads, **Then** that item is opened and scrolled into view.
- [x] **Given** Google Rich Results Test runs on `/`, **When** evaluated,
      **Then** the FAQ schema passes with zero errors.
- [x] **Given** the user navigates with `Tab`, **When** focus lands on a
      summary, **Then** the focus ring is visible and `Enter` toggles.

## 9. AI Prompt (Seed)

> Build `components/FAQ.jsx` + CSS Module. Props: `{ faqs: Array<{q,a}> }`.
> Use native `<details>`/`<summary>`. Must satisfy §6 and §7. The visible Q
> and A must be EXACTLY the strings passed in — no transformation or
> prettifying (they are mirrored into JSON-LD).

## 10. Out of scope

- Search within FAQ.
- "Was this helpful?" feedback.
- Dynamic loading of FAQs from a remote source.
